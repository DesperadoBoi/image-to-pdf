# AGENTS

## Контекст продукта

- приложение конвертирует изображения в PDF;
- приложение offline-first и privacy-first;
- документы не отправляются на сервер;
- текущий MVP описан в [PRODUCT.md](PRODUCT.md);
- архитектура описана в [ARCHITECTURE.md](ARCHITECTURE.md).

## Обязательные действия перед началом любой задачи

1. Прочитать `AGENTS.md`.
2. Прочитать относящиеся к задаче разделы `PRODUCT.md` и `ARCHITECTURE.md`.
3. Выполнить `git status`.
4. Проверить текущую ветку.
5. Изучить существующий код перед созданием новых классов.
6. Составить короткий план.
7. Не менять файлы вне области задачи без необходимости.

## Технологические ограничения

- production-код пишется на Java;
- UI создаётся через XML Views;
- не добавлять Kotlin;
- не добавлять Jetpack Compose;
- не добавлять серверную часть;
- не добавлять Firebase;
- не добавлять аналитику;
- не добавлять рекламу;
- не добавлять DI-фреймворк;
- не менять версии SDK, Gradle, Android Gradle Plugin или Java без прямого требования задачи;
- не добавлять production-зависимости без объяснения;
- не использовать абсолютные пути Android-файлов;
- работать с `Uri` и `ContentResolver`;
- не декодировать одновременно все полноразмерные `Bitmap`;
- не выполнять тяжёлую обработку в UI-потоке.

## Правила изменения кода

- изменения должны быть минимальными и тематически связанными;
- не выполнять массовый рефакторинг попутно;
- не переименовывать несвязанные классы;
- не форматировать весь проект ради одной задачи;
- не дублировать существующую логику;
- использовать понятные английские названия классов и методов;
- комментарии добавлять только там, где они объясняют неочевидное решение;
- пользовательские строки помещать в resources;
- не хардкодить пользовательский текст в Java;
- не хранить секреты, токены и локальные пути в репозитории;
- не изменять `local.properties`;
- не изменять файлы build output;
- не выполнять `git reset --hard`;
- не использовать `force-push`;
- не работать напрямую в `main`;
- не выполнять `commit`, `push` или `merge`, если это явно не разрешено текущим промптом.

## Проверки после изменения production-кода

```powershell
.\gradlew.bat assembleDebug
.\gradlew.bat test
.\gradlew.bat lint
```

Если часть проверок невозможно выполнить, агент обязан явно сообщить причину.

## Формат итогового отчёта

- что изменено;
- какие файлы изменены;
- какие решения приняты;
- какие команды проверки выполнены;
- результат каждой проверки;
- известные ограничения;
- `git status`;
- `commit` и `push` - только если они были разрешены.
