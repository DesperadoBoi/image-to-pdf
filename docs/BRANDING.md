# Branding ImageToPDF

## Основа бренда

ImageToPDF — спокойный и предсказуемый локальный инструмент для преобразования фотографий
и других изображений в PDF. Бренд опирается на четыре свойства продукта:

- простота основного сценария;
- полезный функционал без перегруженности;
- отсутствие рекламы;
- локальная обработка без отправки документов на сервер.

Короткое Android-название для launcher, recent apps и splash — `ImageToPDF`. Название для
будущей карточки Google Play — `ImageToPDF — фото в PDF`. Package и application ID не
являются частью отображаемого бренда и не переименовываются.

## Знак

Основной знак — крупные геометрические буквы `PDF`. Внутреннее пространство `D` работает
как минимальная пиктограмма изображения: один круг обозначает солнце, две связанные
диагональные формы — горы. Так знак связывает исходное изображение и итоговый PDF без
отдельного документа, стрелки или подписи.

Буквы построены как vector paths. Знак не зависит от runtime-шрифта, TextView, Canvas text
или стороннего font-файла. В малом размере сначала должны читаться `PDF`; внутренний символ
может визуально упрощаться, но не должен превращаться в набор мелких штрихов.

## Палитра

| Роль | Цвет | Связь с интерфейсом |
|---|---|---|
| Navy light | `#152E63` | спокойное развитие тёмно-синих поверхностей приложения |
| Navy dark | `#081A3B` | близок к night background и camera surfaces |
| Mark main | `#E8EEFA` | холодный светлый текст на тёмном фоне |
| Mark shade | `#AFC6ED` | мягкий сине-серый tonal переход |
| Image accent | `#7798D7` | приглушённое продолжение violet-blue accent |

Допустим только мягкий tonal gradient. Сильный glow, glassmorphism, тяжёлый 3D,
металлическая текстура, неон и кислотные акценты не используются.

## Safe zone и размеры

Master viewport adaptive icon — `108×108`. Критический знак занимает центральную область
`x=21…87`, `y=34…74`; ни одна значимая деталь не попадает в углы. Эта геометрия должна
оставаться целой под circle, squircle, rounded-square и teardrop masks.

Для самостоятельных экспортов вокруг знака сохраняется как минимум 16% свободного поля.
Не следует увеличивать знак ради заполнения углов. При проверке в `48dp`, `36dp` и `24dp`
приоритет отдаётся читаемости букв; круг и горы оцениваются как вторичная деталь.

## Adaptive launcher icon

- background layer заполняет весь viewport переходом от Navy light к Navy dark;
- background не содержит собственного скруглённого квадрата или рамки;
- foreground layer содержит только `PDF` и символ изображения внутри `D`;
- системный launcher самостоятельно применяет маску устройства;
- внешний текст, второй документ, стрелки, тени и рамки запрещены.

## Monochrome icon

Android 13+ использует отдельный одноцветный vector foreground. Он повторяет геометрию
`PDF`, круга и гор, не содержит background plate, градиента или тени. Цвет задаёт системная
тема launcher, поэтому знак нужно проверять на светлом и тёмном wallpaper.

## Legacy launcher icons

Для Android до adaptive icons используются отдельные растровые fallback assets для `mdpi`,
`hdpi`, `xhdpi`, `xxhdpi` и `xxxhdpi`. Каждый файл экспортируется в своём физическом
размере; один большой bitmap не переиспользуется во всех density. Обычный fallback имеет
полный navy background, round fallback — прозрачные области за пределами круга.

## Splash screen

Splash использует Navy dark background и центральный цветной foreground-знак. Белая
подложка, текст и пользовательская долгая анимация отсутствуют. Android 12+ получает знак
через platform SplashScreen theme attributes; Android 7–11 используют статичный
`windowBackground`. После старта Activity переключается на основную DayNight-тему.

## Запрещённые варианты

- красная конкурентная PDF-стилистика;
- стрелки преобразования;
- документ как отдельный объект;
- внешний rounded-square внутри adaptive layers;
- слово `ImageToPDF` внутри иконки;
- лишние блики, декоративное свечение и мелкие детали;
- использование исходного PNG как launcher foreground без векторного построения.
