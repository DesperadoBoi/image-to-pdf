# PDF export flow

## Экран параметров

`EditorFragment` открывает `PdfExportSheet` до системного выбора файла. Sheet использует уже подключённый Material `BottomSheetDialogFragment`, XML Views и activity-scoped `DocumentSessionViewModel`; новая dependency не добавлена.

Основной вид использует progressive disclosure: постоянно видны только drag indicator, заголовок, закрытие, полноширинное поле имени, компактный выбор качества, свёрнутая строка дополнительных параметров и одна основная кнопка «Конвертировать». Размер страницы, ориентация и поля не показываются одновременно: строка раскрывает три setting rows, каждая из которых открывает небольшой single-choice dialog.

`PdfExportDraft` хранит незавершённый пользовательский ввод и переживает recreation. `PdfExportRequest` создаётся только после валидации и является immutable. `PdfFileNameFormatter` очищает недопустимые символы, отклоняет пустое имя, ограничивает чрезмерно длинное имя и добавляет ровно одну `.pdf`. Расширение показано suffix-ом, а editable часть остаётся однострочной с horizontal scrolling.

## Реальные параметры

- качество: компактные варианты «Компактный», «Баланс» и «Высокое», за которыми остаются реальные `COMPACT` 96 DPI, `BALANCED` 144 DPI и `HIGH` 216 DPI;
- размер страницы: A4 или по изображению;
- ориентация: автоматически по странице, книжная или альбомная;
- поля: без полей, маленькие или стандартные;
- placement остаётся реальным существующим `FIT`, потому что отдельного выбора placement в текущем sheet нет.

`PdfQualityProfile` является единственной точкой значений DPI. `RasterTargetCalculator` принимает DPI профиля, сохраняет исходный aspect ratio и никогда не увеличивает маленькое изображение. `PdfPageLayoutCalculator` применяет orientation к A4 и image-sized страницам. `PdfGenerator` по-прежнему обрабатывает одну страницу за раз и не изменяет исходные файлы.

В основном интерфейсе числовой DPI не показывается. Под выбранным quality chip остаётся одно динамическое пользовательское описание. `PdfExportSummary` формирует строку вида «По изображению · Авто · Стандартные поля» для свёрнутого блока.

## Output Uri

Отдельной кнопки «Обзор» нет. «Конвертировать» использует уже выбранный `Uri`; если его нет, запускает `CreateDocument(application/pdf)` с нормализованным именем и автоматически продолжает генерацию после успешного выбора. Если draft уже содержит output `Uri`, sheet показывает компактную строку «Сохранение: …» и текстовое действие «Изменить».

Cancel системного диалога оставляет sheet открытым и сохраняет draft без status/toast. После начала генерации sheet закрывается, Editor показывает существующие progress и cancel. `operationId`, stale-callback filtering, `CancellationToken` и удаление partial output сохранены. После cancel/error параметры остаются для повтора, а удалённый partial `Uri` очищается из draft. Success оставляет пользователя в Editor и публикует отдельный одноразовый event для banner; подробности находятся в [PDF_RESULT_SCREEN.md](PDF_RESULT_SCREEN.md).

В sheet нет пароля, шифрования, подписи и других неработающих параметров.
